# bsl-pipeline — локальная генерация кода 1С (BSL) под управлением Claude Code

Внутренний пайплайн: клиент даёт ТЗ → ты реализуешь функции 1С быстрее и дешевле → отдаёшь.
Цель — **надёжный результат при экономии лимита подписки** (не API).

## Главное ограничение

**Никакого Claude API.** «Opus» = **Claude Code на подписке** (интерактивно, в этом окружении).
Объёмную генерацию BSL делает **локальная модель** на твоём GPU (бесплатно), чтобы не выжигать лимит подписки.
Дообучение — локально (QLoRA). Полный отчёт и решения: `~/.claude/plans/nifty-drifting-lecun.md`.

## Архитектура

```
ТЗ-кусок
  → Claude Code (подписка): план + RAG-грунт (имена объектов конфигурации)
  → bslgen.py: Ollama (qwen3.5:9b) генерит BSL по плану
  → гейт: BSL Language Server (если подключён) или структурный fallback
        ├─ OK   → код + лог пары (план→код) в data/pairs.jsonl   ← датасет для дообучения
        └─ FAIL → локальный ремонт по тексту ошибки (≤2)
                     └─ всё ещё FAIL → Claude Code дописывает сам (эскалация)
  вне среза (интеграции/сложная логика) → Claude Code напрямую
```

**Срез локали (v1):** запросы/выборки, формы/CRUD, проведение/регистры. Остальное — на Claude Code.

## Использование

```bash
# план строкой + грунт строкой
python bslgen.py --category zapros \
  --plan-text "Функция ... : <план реализации>" \
  --context-text "РегистрНакопления.X — измерения: ...; ресурс: ..."

# план из файла / stdin; грунт из файла (как отдаёт mcp-1c)
python bslgen.py --category formy --plan plan.txt --context context.txt --out Module.bsl
echo "<план>" | python bslgen.py --category provedenie --plan -

# проверка гейта без модели/сети
python bslgen.py --selftest
```

Переменные окружения: `BSLGEN_MODEL` (по умолч. `qwen3.5:9b`), `OLLAMA_URL`, `BSL_LS_JAR`.

## Данные (маховик)

Каждый запуск пишет строку в `data/pairs.jsonl`: `{plan, context, code, gate_ok, gate_errors, category, ...}`.
Для дообучения берём **только `gate_ok: true`** — железное правило «не учим на непроверенном коде».

Обучающий корпус — `data/dataset.jsonl` (chat-формат, всё прошло гейт). Сейчас **101 пара**
(zapros 39 / formy 40 / provedenie 22): 24 рукописных (`seed_data.py` → `build_dataset.py`) +
77 синтетических (`gen_synthetic.py`, пакетный гейт одним прогоном BSL LS, идемпотентно с дедупом).
Расширять дёшево — добавлять шаблоны/наборы параметров в `gen_synthetic.py` и перезапускать.
