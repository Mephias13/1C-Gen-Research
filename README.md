# bsl-pipeline — жизнеспособность малых LLM для генерации кода 1С (исследование)

Эмпирическое исследование: **может ли малая локальная модель (≤4B) экономически
выгодно генерировать рабочий код 1С:Предприятие (BSL)** — и как она соотносится с
большими моделями. Качество проверяется не статическим линтером, а **исполнением кода
в реальной 1С** на публичном бенчмарке.

> **Краткий ответ:** на сегодня — нет. Малая модель (Qwen3-4B, с дообучением и без)
> даёт **0% исполняемо-верных решений**, тогда как флагманская модель — **60%**.
> Дообучение на синтетике и усиление промпта executable-результат **не сдвинули**.
> Коммерческая ценность — в оркестрации сильной модели, а не в локальной 4B.

---

## Гипотеза и мотивация

Идея: клиент даёт ТЗ → пайплайн дёшево генерирует функции 1С локально, дорогую модель
бережём для сложного. Ставка — на **систему, а не на голую модель**:

```
ТЗ → план + RAG-грунтовка (реальные имена объектов конфигурации)
   → локальная модель (Ollama) генерирует BSL
   → гейт: BSL Language Server (компиляторная проверка)
        OK   → код + пара (план→код) в датасет дообучения
        FAIL → локальный ремонт по тексту ошибки (≤2) → иначе эскалация
   → (опционально) QLoRA-дообучение на собранном датасете
```

Компоненты: `bslgen.py` (оркестратор+гейт), `fewshot.py` (RAG few-shot, nomic-embed),
`gen_synthetic.py`/`seed_data.py`/`build_dataset.py` (датасет), `train.py` (QLoRA, unsloth),
`export_gguf.py` (LoRA→GGUF для Ollama).

## Методология

- **Бенчмарк: 1C Code Bench** (147 задач, авторы — **Сбер / GigaCode**, см. благодарности).
  Каждая задача = ТЗ + контекст конфигурации; решение **компилируется и исполняется в
  настоящей 1С**, ответ сверяется с эталоном (Compile Rate / Success Rate).
- **Железо:** RTX 4060 8 ГБ — отсюда потолок на размер локальной модели.
- Сравнение на **одном и том же срезе задач**.

## Результаты

Исполняемый прогон на едином срезе из 30 задач (success = код скомпилировался,
запустился и вернул верный ответ):

| Модель | Compile | Success |
|---|---|---|
| Qwen3.5-4B (база, без обучения) | 17% | **0%** |
| Qwen3-4B + QLoRA (дообучена на нашем датасете) | ~14% | **0%** |
| Qwen3-4B + усиленный grounding-промпт | 10% | **0%** |
| **Флагманская модель (SOTA), эталон-потолок** | **77%** | **60%** |

Сырые метрики и решения — в [`results/`](results/).

## Ключевые выводы

1. **Малая модель не решает 1С исполняемо.** 4B-модель даёт ~0% Success независимо от
   дообучения и промптинга. Это согласуется с публичным лидербордом 1C Code Bench
   (модели ≤35B — единицы %; даже флагманы <52%).
2. **Статический гейт обманчив.** Дообученная модель проходила BSL LS на 67%+ (красивый
   канонический синтаксис), но исполняемый Compile оставался ~14%, Success — 0%. Статанализ
   **сильно переоценивает** качество; правда — только исполнение.
3. **Дообучение на синтетических шаблонах не помогает исполняемости.** Оно учит форме и
   синтаксису, но не **привязке к конкретной задаче** — модель пишет типовые имена полей
   и переименовывает функцию вместо требуемой сигнатуры. Реальная 1С это ловит.
4. **Grounding-промпт тоже не сдвинул** (10% vs 17% — в пределах шума). Барьер не
   в промптинге, а в фундаментальном потолке 4B на доменной логике.
5. **Потолок (SOTA) — 60% Success.** Разрыв с локальной 4B (0%) — качественный, не
   количественный.

## Экономический вывод

Стратегия «дешёвая локальная модель снимает основной объём работы» на текущем поколении
малых моделей **не окупается для исполняемого кода 1С**: полезный offload ≈ 0. Рентабельная
конфигурация — **сильная модель как исполнитель/оркестратор**, а локальные/дешёвые модели —
максимум на тривиальном срезе под надзором. Исследование сохраняет воспроизводимый
пайплайн (сбор данных → дообучение → экспорт → исполняемая оценка) для пере-проверки на
будущих, более способных малых моделях.

## Воспроизведение

```bash
python bslgen.py --selftest                          # самопроверка гейта
python build_dataset.py && python gen_synthetic.py   # датасет (гейтится BSL LS)
# дообучение (WSL2 + CUDA): python train.py ; экспорт: python export_gguf.py
```
Оценка — через 1C Code Bench (репозиторий Сбера; см. благодарности). Бинарники
(BSL LS, JRE, веса моделей, GGUF) в репозиторий не входят.
